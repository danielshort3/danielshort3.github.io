# Short Links redesign concept prompt set

Generated with the built-in image generation tool from the live production Short Links page as a visual reference.

## Shared visual brief

- High-fidelity, implementation-ready desktop UI at a 1440 x 900 landscape composition.
- Preserve the Daniel Short site identity and compact top navigation.
- Use white and `#F9F9FA` canvas, `#091F3B` text, `#005FED` primary actions, `#0145C8` links, `#EEF2F7` surfaces, restrained `#D97706` major-section accents, and `#166534` healthy states.
- Use Inter for interface text and IBM Plex Mono for URLs, codes, and metrics.
- Represent the real workflows: All links, Create link, Templates, project sync, admin/backend health, search/filter/sort, redirects table, and link actions.
- Collapse authenticated token controls to a compact unlocked status.
- Avoid a giant hero, a redundant current-mode card, nested card soup, pill overload, glassmorphism, dark mode, neon styling, and decorative illustrations.

## Concept prompts

1. **Table-First Console**: compact title/status bar, one KPI strip, integrated three-mode switch, single command toolbar, and a full-width redirects table that dominates the viewport.
2. **Master-Detail Workbench**: searchable compact link list on the left and a persistent selected-link inspector with click trend, activity, metadata, and contextual actions on the right.
3. **Command Center Rail**: slim left application rail for Overview, All links, Create link, Templates, Project sync, health, and export; open table-focused canvas on the right.
4. **Focused Three-Mode Studio**: Create link active in a prominent segmented switch; guided Destination, Short code, and Expiration sections with a restrained live preview panel.
5. **Health-First Operations Board**: actionable warning queue, compact health strip, redirects table, and a narrow quick-actions/backend-health panel.
