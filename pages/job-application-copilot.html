Chrome extension · local-first application assistant

Prepare grounded answers. Keep the final decision yours.

Job Application Copilot uses your encrypted evidence and a local Ollama model to prepare cited answers, then requires your review before it fills anything.

Operating boundary

Model

Your encrypted local vault
Filling
Reviewed fields only

The extension is a review assistant. It does not represent that an employer or ATS permits automation.

Evidence-first workflow

From source material to a reviewed field

Every generated proposal stays connected to the material that supports it. Missing evidence becomes a question for you, not an invented answer.

  1. 01

    Capture the job and evidence

    Analyze a posting to capture its visible job description automatically, use only text you explicitly select, or paste a correction from the collapsed context options. Import resumes and reviewed notes; save your Website and exact custom links; and add custom thoughts such as a salary expectation. The encrypted application context follows you to a separate ATS form.

  2. 02

    Review one compact preflight

    Copilot retrieves application-scoped evidence, prepares bounded answer batches locally, and groups verified, review-sensitive, copy-only, needs-input, and manual work before you open individual citation cards.

  3. 03

    Fill deliberately

    Exact identity and link fields come only from facts you verified. Review consequential drafts once, then fill every ready answer or verified answers only. Clarifications, regeneration, filtering, and tracker transfer remain collapsed until you need them.

Setup

Run the model on your computer

Copilot has no cloud-model credential screen. Generation and embeddings use the Ollama service running on this device, with cloud features disabled. Before vault, Ollama, and page handlers become available, you must acknowledge the current versioned data-use disclosure.

Having trouble with setup? Open support.

Browser and runtime

  • Google Chrome 120 or newer.
  • Ollama 0.32.0 or newer at http://127.0.0.1:11434.
  • Keep Ollama bound to loopback rather than a remote network interface.
  • Copilot rejects cloud models and remote model targets.
  • Complete the blocking, versioned first-use acknowledgement.

Default local models

  • qwen3.5:27b for primary generation.
  • qwen3:8b as an optional fallback.
  • nomic-embed-text for local retrieval embeddings.

Allow only the installed extension origin

Disable Ollama cloud features, then allow the exact origin shown in Copilot Settings. The Chrome Web Store ID can differ from a development build, so replace the placeholder below with the ID shown after installation.

OLLAMA_NO_CLOUD=1
OLLAMA_ORIGINS=chrome-extension://<EXTENSION_ID>

Safety boundaries

Designed to stop where judgment matters

The page runtime is intentionally narrow. Copying remains available when safe filling is paused, but the extension does not take over the application.

CAPTCHA pauses filling

If a CAPTCHA is detected, all page mutation pauses until you complete it and rescan. Copilot never solves or bypasses CAPTCHA.

Sensitive decisions stay manual

Consent, signature, demographic, identity, security-code, hidden, upload, and other excluded fields are not actionable Copilot cards.

Optional U.S. work-authorization and sponsorship choices use deterministic local rules; citizenship, immigration/visa status, I-9/E-Verify, export-control, government-ID, and protected demographic questions remain manual and outside Ollama.

Stale pages cannot be filled

A changed URL, DOM revision, or field fingerprint blocks the old proposal until the page is analyzed again.

Submission is always yours

Copilot never clicks submit, accepts terms, signs, consents, or sends a job application. Review the employer and ATS rules before use.

Optional tracker handoff

Carry reviewed job details into your tracker

After analysis, you can explicitly open the account-backed Job Application Tracker with approved job metadata and, if selected, one resume and one cover letter.

  • The tracker asks you to sign in and review the capture.
  • Original file bytes are requested only after that review step.
  • Research, questions, generated answers, custom links, custom thoughts, compensation preferences, and unrelated vault records are excluded.
  • Nothing is stored until you select Save application in the tracker.

Support

Questions, setup trouble, or a site that needs an adapter?

Describe what happened without sending resumes, completed form values, or other personal application data. A sanitized ATS fixture is the preferred diagnostic.
