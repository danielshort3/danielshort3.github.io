GA4 UTM Performance - Private UTM reporting via the GA4 Data API.

GA4 UTM Performance

Workflow

Run reports in 3 steps

  1. Access: Save your admin token and choose a property profile.
  2. Scope + Filters: Set a date range and optional UTM constraints.
  3. Run: Generate UTM URL or Explore reports, then export CSV.
Step 1

Access

Store an admin token on this device and save GA4 properties as reusable profiles.

Stored in localStorage on this device.

Store multiple GA4 properties and switch quickly.

Server env: GA4_ADMIN_TOKEN, GA4_SERVICE_ACCOUNT_JSON_B64, and (optionally) GA4_PROPERTY_ID.

Step 2

Scope

Choose a report window. Wider ranges help when campaign attribution is delayed.

Tip: GA4 can take time to attribute new campaigns. If you’re not seeing results, try a wider date range.

Step 3

Filters

Filter by specific UTM values or keep broad matching to inspect all tagged traffic.

Applies to the URL report (filters pageLocation). Insights use GA4 session traffic-source dimensions.

Step 4

UTM URL Report

Parse URL-level traffic into UTM groups, then pivot, filter, and drill into specific combinations.

Groups are created client-side by parsing pageLocation.

Step 5

Explore

Run custom GA4 dimension/metric cuts, then segment rows into guided drilldown views.

These are GA4 Data API dimension names (examples: country, deviceCategory, sessionSource).

Tip: choose fewer metrics if GA4 rejects a combination.
